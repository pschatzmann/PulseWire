using ir_transceiver::TransceiverKaseikyo = typedef TransceiverTemplate<KaseikyoPreamble, PulseDistanceCodec, 37900, 3, RxDriverGeneric, TxDriverGeneric>

Kaseikyo protocol (Panasonic, JVC, Denon, Mitsubishi, Sharp, etc.)

  • Modulation: Pulse Distance
  • Carrier Frequency: 37.9 kHz
  • Bit Encoding: Logical '1' = 432us mark + 1296us space, Logical '0' = 432us mark + 432us space
  • Frame: 3456us leading mark + 1728us space (handled by KaseikyoPreamble)
  • Address: 16 bits, Command: 8 bits, etc. (varies by device)
  • Used in Panasonic and other Japanese brands

NEC protocol (NEC, NEC1, NEC2, Apple, Onkyo, etc.)

  • Modulation: Pulse Distance
  • Carrier Frequency: 38 kHz
  • Bit Encoding: Logical '1' = 560us mark + 1690us space, Logical '0' = 560us mark + 560us space
  • Frame: 9ms leading mark + 4.5ms space (handled by NECPreamble)
  • Address: 8 bits, Command: 8 bits, Inverse: 8 bits each (32 bits total)
  • Widely used in consumer electronics (TV, audio, etc.)

◆ TransceiverSony

using ir_transceiver::TransceiverSony = typedef TransceiverTemplate<SonyPreamble, PulseDistanceCodec, 40000, 3, RxDriverGeneric, TxDriverGeneric>

Sony SIRC protocol (Sony TV, DVD, HiFi, etc.)

  • Modulation: Pulse Distance
  • Carrier Frequency: 40 kHz (often 38 kHz works)
  • Bit Encoding: Logical '1' = 1200us mark + 600us space, Logical '0' = 600us mark + 600us space
  • Frame: 2.4ms leading mark (handled by SonyPreamble)
  • Address: 5 bits, Command: 7 bits (12, 15, or 20 bits total)
  • Used in Sony remotes (TV, DVD, etc.)
