Detailed Description

Header-only Arduino class for Manchester IR decoding using interrupts.

This class decodes Manchester-encoded IR signals using hardware interrupts. It works on both ESP32 and RP2040 platforms using the Arduino API. Features:

  • Interrupt-driven decoding for reliable timing
  • Manchester bit decoding with configurable RX pin
  • Byte boundary detection using a preamble (default: 0xAA)
  • Checksum validation for received data
  • Callback mechanism for decoded data
  • Header-only implementation for easy integration

Maximum Supported Speed:

  • The default timing window allows a maximum bit rate of about 5 kbit/s (200 µs per bit).
  • For reliable operation, speeds up to 2–3 kbit/s are recommended, depending on hardware and signal quality.
  • You can adjust the timing window in the code for your application and hardware.
Warning
On platforms without attachInterruptArg, only 10 instances of RxDriverGeneric can be active due to ISRManager limitations. Ensure that each instance uses a unique RX pin and that the total number of active instances does not exceed 10 to avoid conflicts.
