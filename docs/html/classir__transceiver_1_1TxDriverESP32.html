Detailed Description

High-performance ESP32 IR TX driver using the RMT peripheral and Manchester encoding.

This class converts outgoing bytes to Manchester-encoded RMT items and transmits them using the ESP32 RMT hardware. Supports carrier frequency, duty cycle, and frame size configuration.

Bit frequency (Hz) determines the baud rate for IR transmission. Realistic values for reliable IR communication are typically:

  • 300 to 2400 Hz for most consumer IR remotes
  • 1000 to 2000 Hz for robust, error-free transmission in Arduino/ESP32 projects
  • Higher frequencies (up to 4000 Hz) are possible with ESP32 RMT, but may require careful timing and receiver tuning

For Manchester encoding, bit frequency = baud rate (bits per second).

For Pulse Distance and Pulse Width protocols:

  • Bit frequency is the rate at which pulses (or pulse/space pairs) are generated.
  • Baud rate is determined by the average bit duration (pulse + space or pulse width).
  • Typically, baud rate < bit frequency, since each bit may require a longer or variable time.
  • Bit frequency sets the timing reference for pulse generation, but the actual data rate depends on the protocol's encoding scheme.

Summary:

  • Manchester: bit frequency = baud rate.
  • Pulse Distance/Pulse Width: baud rate = 1 / average bit duration; bit frequency is the timing base for pulse encoding.
Note
Supports sending and receiving on the same microcontroller without conflicts.
